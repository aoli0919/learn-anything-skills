## Daily Loop Template

Use this for every day after the first week:

1. Retrieval practice, 5-10 minutes:
   - answer one old check question without notes
2. New task, 25-60 minutes:
   - read, build, explain, or debug one small thing
3. Visible artifact, 10-30 minutes:
   - concept card, diagram, toy script, table, or learning log
4. Check question:
   - one question that proves whether the idea can be used
5. Reflection:
   - what is still fuzzy?
   - should tomorrow continue, slow down, review, or build?

## Adjustment Rules

- If the same concept is fuzzy for two sessions, slow down and use a worked example.
- If setup fails for more than 30 minutes, switch to the fallback project.
- If the check question is easy, move to a small build.
- If a project fails, log the failure and reduce the next task by half.
- Do not add more than three new resources per week.
